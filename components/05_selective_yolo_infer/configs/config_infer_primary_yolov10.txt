# YOLOv10s primary detector for P0.5.
#
# This config is the production target for "selective YOLO inference" — it
# pulls in a real YOLOv10s ONNX plus a custom bbox parser library that knows
# how to decode YOLOv10's NMS-free output. Both files are external assets
# staged by scripts/fetch_models.sh.
#
# Why YOLOv10 specifically (not v8): we already have a working v10s ONNX +
# parser .so in a sibling project (DeepstreamPOC/models/yolo/), licensed and
# already verified against this exact DeepStream 9.0 nvinfer ABI. Reusing it
# costs nothing and avoids re-building a parser from scratch. If you swap in
# YOLOv8 or a newer variant, you also need to swap the parser .so.
#
# Required staged files under models/yolo/ (created by fetch_models.sh):
#   - yolov10s.onnx
#   - yolov10s.onnx.data         (external weights for the ONNX)
#   - libnvdsinfer_custom_impl_Yolo.so
#   - labels.txt                 (COCO 80 class names)
# The TRT engine model.engine is regenerated on first run inside this dir.
