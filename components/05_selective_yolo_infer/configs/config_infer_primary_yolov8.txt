# YOLOv8 slot for P0.5.
#
# This file is intentionally a TEMPLATE. nvinfer cannot decode raw YOLOv8 ONNX
# output without a custom bbox-parser library (the standard YOLOv8 head shape
# [batch, 84, 8400] does not match nvinfer's built-in parsers). The accepted
# in-tree path on DS9.0 is:
#
#   1. Stage a YOLOv8 ONNX at models/yolov8n.onnx (see scripts/fetch_models.sh).
#   2. Build a custom parser library, either:
#        - the marcoslucianops/DeepStream-Yolo parser
#          (https://github.com/marcoslucianops/DeepStream-Yolo, MIT-licensed),
#          built into models/libnvdsinfer_custom_impl_Yolo.so; or
#        - export the ONNX with EfficientNMS_TRT fused, in which case the
#          built-in parser works but the export tooling lives outside this
#          component.
#   3. Point parse-bbox-func-name + custom-lib-path below at that build.
#
# Until step 2 is done, do *not* pass --infer-config configs/config_infer_primary_yolov8.txt
# at runtime; the default TrafficCamNet config is the supported zero-friction
# path. This file exists so the swap is one config change instead of a code
# refactor.
